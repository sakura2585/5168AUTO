# 5168AUTO 使用者提示文

> 5168 房仲後台（[007.houseprice.tw](https://007.houseprice.tw/)）庫存自動化工具  
> 目前版本：**v0.2.19**（以程式標題列為準）

---

## 一、這支程式做什麼？

5168AUTO 可協助您：

1. **登入** 5168 後台（手機＋簡訊驗證碼，Session 會保存在本機）
2. **自動執行庫存修改流程**（7 個步驟，完成後可無限循環）
3. **循環時自動關閉廣告彈窗**（可自訂 XPath）

適用情境：需要反覆對庫存最後一頁、最後一筆物件執行「修改 → 完成 → 返回庫存」的使用者。

---

## 二、使用前準備

| 項目 | 說明 |
|------|------|
| 作業系統 | Windows 10 或以上 |
| 瀏覽器 | 已安裝 **Google Chrome**（必須） |
| 網路 | 可連線 5168 與簡訊驗證 |
| 5168 帳號 | 手機號碼可接收驗證碼 |

**不需**手動下載 ChromeDriver，程式會自動配對。

---

## 三、下載與安裝

1. 向管理員取得 **GitHub Releases** 下載連結  
   （若倉庫為私人，需先被加入 Collaborator 才能下載）
2. 進入 Releases 頁，下載 **`5168AUTO_v版號.zip`**  
   ⚠️ 請下載 **Assets 裡的 zip**，不要下載 Source code
3. 解壓縮到任意資料夾（例如 `D:\Tools\5168AUTO\`）
4. 雙擊 **`5168AUTO_v版號.exe`** 即可執行

首次執行會在 exe 同目錄建立 `data\` 資料夾（設定、登入 Session）。

---

## 四、建議操作流程（第一次）

### 步驟 1：填寫基本設定

1. **手機號碼**：您的 5168 登入手機
2. 勾選 **記住手機號碼 / Remember phone**
3. 按 **儲存 / Save**

### 步驟 2：登入

1. 按 **開啟瀏覽器登入 / Open Browser Login**
2. Chrome 會自動開啟，程式會預填手機並嘗試觸發驗證碼
3. 在 Chrome 輸入 **簡訊驗證碼** 完成登入
4. 日誌出現「已登入」即成功

### 步驟 3：執行庫存流程

若已勾選 **登入後自動執行庫存流程**，登入成功後會自動開始。

也可手動按 **執行庫存流程 / Run Workflow**（需已登入）。

---

## 五、庫存流程說明（7 步 + 循環）

| 步驟 | 動作 |
|------|------|
| 1 | 尋找並展開頂部選單 |
| 2 | 點擊子選單（進入庫存相關頁） |
| 3 | 點擊頁數列 **最後一頁** |
| 4 | 點擊列表 **最後一筆「修改」**，開新分頁後關閉其他分頁 |
| 5 | 點擊 **「我已詳閱並了解」** |
| 6 | 點擊 **「完成」** |
| 7 | 點擊 **「返回庫存」** |
| 循環 | 預設回到步驟 3 繼續，直到您按 **停止 / Stop** |

---

## 六、介面設定說明

### 登入設定

| 欄位 | 說明 | 建議值 |
|------|------|--------|
| 啟動網址 | 5168 首頁 | 預設即可 |
| 手機號碼 | 登入用手機 | 09 開頭 |
| 登入等待逾時 | 等待您完成登入的最長秒數 | 300 秒 |
| 驗證碼等待 | 發送簡訊後的倒數秒數 | 60 秒 |

### 庫存流程

| 欄位 | 說明 | 建議值 |
|------|------|--------|
| 元素等待 | 每步等待網頁元素出現的秒數 | 15 秒 |
| 步驟間隔 | 步驟之間暫停秒數 | 0.8 秒 |
| 登入後自動執行庫存流程 | 登入成功後接續 1～7 步 | 建議勾選 |
| 循環步驟 3～7 | 完成一輪後是否繼續 | 依需求 |
| 最大循環次數 | **0 = 無限**直到按停止 | 0 |
| 循環彈窗 XPath | 每輪回到步驟 3 前點擊關閉廣告 | 見下方 |

### 循環彈窗 XPath（進階）

僅在 **第 2 輪起、回到步驟 3 之前** 會依序嘗試點擊。

常見 5168 公告彈窗可新增：

```
/html/body/div[6]/div[2]/button[1]
```

操作：輸入 XPath → **新增 / Add** → **儲存 / Save**

## 七、按鈕功能

| 按鈕 | 功能 |
|------|------|
| 儲存 / Save | 保存設定與手機號碼 |
| 開啟瀏覽器登入 | 開 Chrome 並引導登入；成功後可自動跑流程 |
| 檢查登入狀態 | 只檢查是否已登入；已登入且勾選自動執行時也會跑流程 |
| 執行庫存流程 | 直接執行 1～7 步（沿用已開啟的 Chrome） |
| 停止 / Stop | 中斷目前登入或流程 |
| 清除 Profile | 刪除本機登入 Session，需重新簡訊驗證 |

## 八、常見問題

### 1. 一直顯示未登入

- 在 Chrome 視窗完成簡訊驗證
- 或按 **清除 Profile** 後重新登入

### 2. 流程跑到某一步就停住

- 5168 網頁可能改版，元素 XPath 需調整
- 查看下方日誌的錯誤訊息
- 可加大 **元素等待** 秒數

### 3. 循環時被廣告彈窗擋住

- 在 **循環彈窗 XPath** 新增「關閉」按鈕的 XPath
- 用 Chrome 開發者工具（F12）複製正確 XPath

### 4. Chrome 沒有出現

- 確認已安裝 Google Chrome
- 關閉其他可能占用同一 Profile 的 Chrome
- 重開程式再試

### 5. 下載 Releases 顯示 404

- 倉庫若為 **Private**，需管理員邀請您為協作者
- 或請管理員改為 Public 倉庫

### 6. 更新版本

1. 關閉舊程式
2. 下載新版 zip 解壓
3. 覆蓋或換新資料夾執行
4. `data\` 資料夾可保留（登入 Session 通常仍有效）

---

## 九、資料存放位置

| 路徑 | 內容 |
|------|------|
| `data\state.json` | 視窗大小、逾時、XPath 等設定 |
| `data\credentials.json` | 手機號碼（若勾選記住） |
| `data\chrome_profile\` | Chrome 登入 Cookie／Session |

⚠️ 請勿將 `data\` 資料夾分享給他人（含登入狀態）。

---

## 十、注意事項

- 執行期間 **請勿手動關閉** 程式開啟的 Chrome 視窗
- 流程進行中可按 **停止**，但 Chrome 會保留不關
- 本工具僅輔助重複操作，請確認 5168 後台操作符合您的業務需求
- 若 5168 網站大幅改版，流程可能需要更新程式版本
